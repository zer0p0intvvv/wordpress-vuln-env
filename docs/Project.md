## 工作流
    1. 环境创建

    vulhub-env/
    ├── base/<plugin-slug>/<version>/
    │   ├── Dockerfile              # wordpress:php7.4-apache + wp-cli
    │   ├── docker-entrypoint.sh    # 自动安装 WordPress + 插件
    │   └── plugins/<slug>/         # 插件源码
    ├── <plugin-slug>/<CVE-ID>/
    │   ├── docker-compose.yml      # web(80xx) + mysql(33xx)
    │   ├── README.md
    │   └── README.zh-cn.md
    └── environments.toml

    2. 版本确认

    # 检查本地插件版本
    grep -m1 "Version:"
    /Applications/MAMP/www/wordpress6.7.1/wp-content/plugins/<slug>/*.php

    # 对比漏洞影响版本（来自 CVE-INFO JSON）
    # 如果本地版本 > 修复版本 → 从 WP.org 下载旧版本
    curl -sL "https://downloads.wordpress.org/plugin/<slug>.<version>.zip" -o
    /tmp/<slug>.zip

    3. 版本不匹配处理

    ┌─────────────────────┬──────────────────────┐
    │        情况         │         处理         │
    ├─────────────────────┼──────────────────────┤
    │ 本地版本 < 修复版本 │ 直接用本地源码       │
    ├─────────────────────┼──────────────────────┤
    │ 本地版本 > 修复版本 │ 从 WP.org 下载旧版本 │
    ├─────────────────────┼──────────────────────┤
    │ WP.org 已移除       │ 跳过（如 gallery）   │
    └─────────────────────┴──────────────────────┘

    4. 插件特殊处理

    ┌──────────────────┬───────────────┬─────────────┬────────────────────────┐
    │       插件       │      CVE      │    问题     │        解决方案        │
    ├──────────────────┼───────────────┼─────────────┼────────────────────────┤
    │ photo-gallery    │ CVE-2022-0169 │ 数据库为空  │ entrypoint 自动创建    │
    │                  │               │             │ gallery + shortcode    │
    ├──────────────────┼───────────────┼─────────────┼────────────────────────┤
    │ booking-calendar │ CVE-2022-3982 │ upload      │ entrypoint             │
    │                  │               │ 目录不存在  │ 自动创建目录           │
    ├──────────────────┼───────────────┼─────────────┼────────────────────────┤
    │ instawp-connect  │ CVE-2024-2667 │ REST API    │ entrypoint 手动 new    │
    │                  │               │ 类未实例化  │ InstaWP_Rest_Api()     │
    └──────────────────┴───────────────┴─────────────┴────────────────────────┘

    5. nuclei 验证

    cd ~/工具

    # 无认证漏洞
    ./nuclei -t ~/nuclei-templates/http/cves/<year>/<CVE>.yaml -u
    http://localhost:<port>

    # 需认证漏洞
    ./nuclei -t ~/nuclei-templates/http/cves/<year>/<CVE>.yaml -u
    http://localhost:<port> \
        -V "username=admin" -V "password=admin"

    6. nuclei 检测失败排查

    ┌─────────────────────┬─────────────────────────┬─────────────────────────┐
    │        现象         │          原因           │          解决           │
    ├─────────────────────┼─────────────────────────┼─────────────────────────┤
    │ No results + 302    │ 需要登录                │ 加 -V "username=admin"  │
    │ 跳转                │                         │ -V "password=admin"     │
    ├─────────────────────┼─────────────────────────┼─────────────────────────┤
    │ No results + 空响应 │ 数据库无数据            │ 插入测试数据            │
    ├─────────────────────┼─────────────────────────┼─────────────────────────┤
    │ No results + AJAX   │ nonce 失败或 action     │ 检查插件版本和 hook     │
    │ 返回 -1             │ 未注册                  │ 注册                    │
    ├─────────────────────┼─────────────────────────┼─────────────────────────┤
    │ interactsh 错误     │ OOB                     │ 手动验证或跳过          │
    │                     │ 检测，容器无法接收回调  │                         │
    ├─────────────────────┼─────────────────────────┼─────────────────────────┤
    │ 上传成功但检测不到  │ upload 目录权限/不存在  │ 修复目录权限            │
    └─────────────────────┴─────────────────────────┴─────────────────────────┘

    7. 批量管理

    cd vulhub-env

    # 启动/停止单个
    cd <plugin>/<CVE> && docker compose up -d --build
    cd <plugin>/<CVE> && docker compose down -v

    # 启动全部
    ./start.sh start
